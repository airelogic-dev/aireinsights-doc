    AireInsights Structure

    May 20, 2021

    The pipeline

    The pipeline is a major component in the AireInsights software structure, handling data inputs, outputs, and configuring the steps and callable modules.

    The schematic below illustrates a possible pipeline configuration. Steps A-F represent independent stages at which a task occurs. The tasks can include filtering data, calling modules or external services.

    Example of json and yaml pipeline configurations are shown below. The Steps value is a list of step configurations which parse data to the analytics modules e.g. using the python SDK.

    // json
{
  Creator: 'testuser',
  Name: 'example_pipeline',
  Steps: [
    {
      InputTemplate: ' {{ input.value }} ',
      ModuleContext: '{"hardcodedResponse":"processed data"}',
      ModuleInputFormat: 2,
      ModuleName: 'example_module',
      ModuleOutputFormat: 2,
      Name: 'example_step'}
  ],
  Output: {
    CallbackEndpoint: 'http://host.docker.internal:5123/',
    DataFormat: 0,
    Template: '{"output_data": " {{ example_step }}" }'
  }
}

    # yaml
Creator: testuser
Name: example_pipeline
Steps:
- InputTemplate:  '{{ input.value }}'  
  ModuleContext: '{"hardcodedResponse":"processed data"}'
  ModuleInputFormat: 2
  ModuleName: example_module
  ModuleOutputFormat: 2
  Name: example_step
Output:
  CallbackEndpoint: http://host.docker.internal:5123/
  DataFormat: 0
  Template: '{"output_data":  "{{ example_step }}"}"
